62 /* Header of a user record. These records are attached by client programs
63 dynamically at runtime. A unique string ID must be given to each record
64 attached to the same object. Cgraph has functions to create, search for,
65 and delete these records. The records are maintained in a circular list,
66 with obj->data pointing somewhere in the list. The search function has
67 an option to lock this pointer on a given record. The application must
68 be written so only one such lock is outstanding at a time. */
